<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mastery_context → system prompt | mastery_context dicts contain topic names from curriculum (trusted static data) and bucket labels (computed internally) — no user-supplied strings |
| api/chat.py → mastery_context_for_prompt | child_id from header (untrusted) passed to service; service scopes DB query to that child_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-13 | Tampering | mastery_context topic names in prompt | accept | Names are sourced from curriculum.py static data (not user input); no injection surface |
| T-2-14 | Information Disclosure | mastery_context exposes child's mastery state in prompt | accept | System prompt is sent to the LLM only; not returned to client; LLM has no cross-child access |
| T-2-15 | Spoofing | child_id header in chat endpoint | accept | Phase 2 has no auth; auth is Phase 4 scope; mastery_context_for_prompt is child-scoped |
</threat_model>
