    update_fsrs_schedule(child_id, kc_id, correct, db, review_datetime=None):
    - After calling with correct=True: mastery_state.stability is a positive float (not None)
    - After calling with correct=True: mastery_state.next_review is a UTC-aware datetime in the future
    - After calling with correct=True: mastery_state.card_state is a non-empty string ("Learning", "Review", or "Relearning")
    - After calling with correct=True: mastery_state.difficulty_d is a positive float (not None)
    - After calling with correct=False (Again rating): mastery_state.next_review is closer in time
      than after correct=True (Again shortens the interval)
    - difficulty_d is stored as card.difficulty (FSRS Card attribute) — NOT card.difficulty_d

    fit_fsrs_params(child_id, db):
    - Given 0 rated events for a child: returns without writing to child_fsrs_params (no row created)
    - Given 9 rated events: returns without writing (below FSRS_MIN_REVIEWS_FOR_FIT = 10)
    - Given 15 rated events: writes a row to child_fsrs_params with weights as a list of floats
    - The written weights list has length 21 (FSRS-5 parameter count)

    Timezone handling:
    - next_review stored in mastery_state may be read back timezone-naive from SQLite
    - All comparisons in this module must normalise: `nr.replace(tzinfo=timezone.utc)` when nr.tzinfo is None
  </behavior>

    4. Implement update_fsrs_schedule():
       Signature: `async def update_fsrs_schedule(child_id: str, kc_id: str, correct: bool,
           db: AsyncSession, review_datetime: Optional[datetime] = None) -> None`
       Steps:
       - Load per-child FSRS weights via get_child_fsrs_params(child_id, db).
         If None: scheduler = Scheduler(). Else: scheduler = Scheduler(parameters=tuple(row.weights)).
       - Load mastery_state row via create_or_get_mastery_state(child_id, kc_id, db).
       - Reconstruct Card from stored fields:
         If mastery_state.stability is None: card = Card() (new card).
         Else: card = Card(); set card.stability, card.difficulty (from mastery_state.difficulty_d),
         card.due (from mastery_state.next_review normalised to UTC-aware if needed),
         card.state = State[mastery_state.card_state] if mastery_state.card_state else card.state.
       - Map correct→Rating: correct=True → Rating.Good; correct=False → Rating.Again
         (per RESEARCH.md "FSRS Rating Mapping" — simple two-level mapping).
       - Call: now = review_datetime or datetime.now(timezone.utc)
         card, _ = scheduler.review_card(card, rating, review_datetime=now)
       - Write back via update_mastery_state(child_id, kc_id, db,
           stability=card.stability,
           difficulty_d=card.difficulty,
           card_state=card.state.name,
           next_review=card.due)
         NOTE: FSRS Card uses card.difficulty (not card.difficulty_d). Store in difficulty_d column.

    5. Implement fit_fsrs_params():
       Signature: `async def fit_fsrs_params(child_id: str, db: AsyncSession) -> None`
       Steps:
       - SELECT all InteractionEventModel rows WHERE child_id == child_id AND correct IS NOT NULL,
         ORDER BY timestamp ASC. Use parameterised select() — never string-interpolated.
       - If len(events) < FSRS_MIN_REVIEWS_FOR_FIT: return (cold-start guard).
       - Reconstruct ReviewLog list from events. For each event:
         map correct=True→Rating.Good, correct=False→Rating.Again.
         Build ReviewLog(rating=rating, review_datetime=event.timestamp.replace(tzinfo=timezone.utc)
         if event.timestamp.tzinfo is None else event.timestamp).
         Import ReviewLog from fsrs.
       - from fsrs import Optimizer
         optimizer = Optimizer(review_logs)
         optimal = optimizer.compute_optimal_parameters()
       - await upsert_child_fsrs_params(child_id, list(optimal), db)
