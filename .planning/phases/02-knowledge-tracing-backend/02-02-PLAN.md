<feature>
  <name>BKT mastery model — KT-01</name>
  <files>services/knowledge_tracing.py, tests/services/test_knowledge_tracing.py, tests/services/__init__.py</files>
  <behavior>
    Pure function update_bkt():
    - correct=True, p_mastery=0.1, p_learn=0.2, p_slip=0.1, p_guess=0.2 → result > 0.1 and result <= 1.0
    - correct=False, p_mastery=0.5, p_learn=0.2, p_slip=0.1, p_guess=0.2 → result < 0.5
    - correct=True, p_mastery=0.99 (near certain) → result stays near 1.0 (does not decrease)
    - correct=False, p_mastery=0.01 (near zero) → result stays near 0.0 (does not spike)
    - Output always clamped to [0.0, 1.0]

    Async update_bkt_for_session(session_id, db):
    - Given a child with two interaction_events for same kc_id (correct=True, correct=True) in one session:
      → mastery_state.p_mastery for that kc_id is higher than default 0.1 after the call
    - Given a session with no kc_id-tagged events (all NULL): returns {} without raising
    - Returns dict {kc_id: new_p_mastery} for all KCs updated in the session
    - Processes events in chronological order per KC (earliest timestamp first)
  </behavior>
