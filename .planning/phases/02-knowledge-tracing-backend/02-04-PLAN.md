  <behavior>
    next_topics(child_id, db, limit=10) → list[Topic]:
    - Given a child with KC-A (next_review=yesterday, p_mastery=0.4) and KC-B (next_review=tomorrow):
      → KC-A appears before KC-B in the result
    - Given KC-X (fragile: p_mastery=0.3, due today) and KC-Y (in_progress: p_mastery=0.8, due today):
      → KC-X appears before KC-Y (fragile ranked before in_progress, D-08)
    - Given KC-Z (solid: p_mastery=0.97, next_review=next week):
      → KC-Z does not appear in result (excluded per D-08)
    - Given a KC with no mastery_state row:
      → it appears in result (treated as not_started / due today, D-09)
    - Result length <= limit
    - Returns [] (empty list) if child has no eligible curriculum topics

    mastery_context_for_prompt(child_id, db, limit=5) → list[dict]:
    - Returns list of dicts each with keys "name" (str) and "bucket" (str)
    - bucket values are one of: "fragile", "in_progress", "not_started"
    - "solid" bucket never appears in the returned list (solid KCs are filtered out)
    - Length <= limit
    - Returns [] when next_topics() returns []
  </behavior>
