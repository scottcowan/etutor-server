      @asynccontextmanager
      async def lifespan(app: FastAPI):
          # Ensure data/ directory exists before SQLite tries to create the file
          Path("data").mkdir(exist_ok=True)
          # Create tables that don't exist (Alembic handles schema evolution;
          # create_all is a startup safety net so dev server works without running alembic)
          async with engine.begin() as conn:
              await conn.run_sync(Base.metadata.create_all)
          # Seed dev profiles — idempotent upserts; skipped in production (D-06)
          settings = get_settings()
          if getattr(settings, "env", "dev") != "production":
              async with AsyncSessionLocal() as session:
                  await seed_dev_data(session)
          yield
          # Shutdown: no cleanup needed for SQLite; dispose engine on exit
          await engine.dispose()

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → api/chat.py route | X-Child-ID and X-Device-ID headers are untrusted user input; validated before DB lookup |
| lifespan → filesystem | Path("data").mkdir creates a local directory; no user input involved |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-13 | Tampering | api/chat.py X-Child-ID / X-Device-ID headers used as DB query params | mitigate | SQLAlchemy ORM parameterised queries in db/crud.py; header values never string-interpolated into SQL |
| T-1-14 | Information Disclosure | lifespan seeds KidA/KidB profiles on every non-production startup | accept | Generic identifiers only (D-08); no PII in seed data; seeds are idempotent INSERT OR IGNORE |
| T-1-15 | Denial of Service | chat endpoint creates a DB session per request | accept | SQLAlchemy connection pool handles concurrency; Phase 1 is single-user dev server |
| T-1-16 | Elevation of Privilege | session parameter is FastAPI Depends — could theoretically be overridden in test | accept | D-10 explicitly uses explicit session arg for evals, not dependency_overrides; no privilege risk |
| T-1-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan |
</threat_model>
