<feature>
  <name>ChildProfile CRUD and Seed Idempotency</name>
  <files>db/crud.py, tests/db/test_crud_profiles.py, tests/db/test_seeds.py</files>
  <behavior>
    - create_child(session, id, name, age, **kwargs) → ChildProfileModel: inserts and returns new row
    - get_child_by_id(child_id, session) → Optional[ChildProfileModel]: returns model or None for unknown id
    - get_child_by_device_id(device_id, session) → Optional[ChildProfileModel]: returns model matching device_id or None
    - list_children(session) → list[ChildProfileModel]: returns all rows
    - update_interests(child_id, new_interests, session): merges new_interests with existing (set union), commits

    Test cases for test_crud_profiles.py:
    - test_get_child_by_id_returns_none_for_unknown: get_child_by_id("nonexistent", session) returns None
    - test_create_and_get_by_id: create then retrieve; name, age, id match
    - test_get_by_device_id: create with device_id="device-test"; get_child_by_device_id returns it
    - test_get_by_device_id_returns_none_for_unknown: get_child_by_device_id("no-such-device", session) returns None
    - test_list_children: create 2 children; list_children returns both
    - test_update_interests_merges: create with interests=["a"]; update_interests with ["b","c"]; get_child_by_id has interests {"a","b","c"} (set)

    Test cases for test_seeds.py:
    - test_seed_is_idempotent: call seed_dev_data(session) twice; SELECT COUNT(*) FROM child_profiles = 2
    - test_seed_kida_profile: after seed, get_child_by_id("child-kida", session).name == "KidA"
    - test_seed_kidb_profile: after seed, get_child_by_id("child-kidb", session).name == "KidB"
  </behavior>
  <implementation>
    db/crud.py imports: AsyncSession from sqlalchemy.ext.asyncio; select from sqlalchemy; ChildProfileModel from db.models; Optional from typing.

    create_child signature: `async def create_child(session: AsyncSession, *, id: str, name: str, age: int, device_id=None, interests=None, reading_level="age-appropriate", current_topic=None, current_books=None, session_count=0, neurodivergence=None) -> ChildProfileModel`
    Body: instantiate ChildProfileModel, session.add(model), await session.commit(), await session.refresh(model), return model.

    get_child_by_id: `async def get_child_by_id(child_id: str, session: AsyncSession) -> Optional[ChildProfileModel]`
    Body: result = await session.execute(select(ChildProfileModel).where(ChildProfileModel.id == child_id)); return result.scalar_one_or_none()

    get_child_by_device_id: same pattern with .where(ChildProfileModel.device_id == device_id)

    list_children: result = await session.execute(select(ChildProfileModel)); return list(result.scalars().all())

    update_interests: get child by id; existing = set(child.interests or []); merged = list(existing | set(new_interests)); child.interests = merged; await session.commit()

    Parameter order note: CONTEXT.md D-03 and PATTERNS.md "After migration" examples consistently use (id_or_string, session) order. Match this exactly so Plan 05's thin wrapper can delegate without reordering.
  </implementation>
</feature>
