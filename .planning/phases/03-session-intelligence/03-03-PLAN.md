    File structure (follow services/knowledge_tracing.py import/docstring/function style):

    Module docstring citing each requirement: HIST-01, HIST-02, CURR-02, CURR-03.

    Imports:
        from collections import defaultdict

    Module-level state (D-07):
        # Module-level session-level turn counter for rubber-band escalation (D-07).
        # Single-worker assumption — see RESEARCH.md Pitfall 5 for multi-worker limitation.
        _prereq_turn_counter: dict[tuple[str, str], int] = defaultdict(int)

    Module-level tag-to-topic index (built once at import, used by extract_interests_from_turns):
        # Tag index: {tag_lower: topic_name} — built once from CURRICULUM (870 topics).
        _tag_to_topic: dict[str, str] = {}
        for _t in CURRICULUM:
            for _tag in _t.tags:
                _tag_to_topic[_tag.lower()] = _t.name

    Function 1 — get_prereq_turn(child_id: str, kc_id: str) -> int
    Function 2 — increment_prereq_turn(child_id: str, kc_id: str) -> int
    Function 3 — reset_prereq_turn(child_id: str, kc_id: str) -> None

    Function 4 — async build_24hr_history_context(child_id: str, db: AsyncSession) -> list[str]:
    - Calls get_24hr_history(child_id, db) to get events (already filtered to 24h DESC)
    - Iterates events; extracts event.topic, skipping None/empty
    - Deduplicates while preserving order (seen set): first occurrence of each topic name wins (= most recent, since DESC)
    - Stops after 8 unique topic names (D-02 token cap)
    - Returns list of topic name strings

    Function 5 — async build_prereq_tree_context(child_id: str, db: AsyncSession, limit: int = 5) -> list[dict]:
    - Calls `await next_topics(child_id, db, limit=limit)` to get candidate topics
    - Gathers all unique prerequisite IDs from candidate topics' `.prerequisites` lists
    - Batch-loads MasteryStateModel rows for those prereq IDs using SQLAlchemy select().where().in_() pattern (see PATTERNS.md batch mastery load pattern)
    - Builds prereq_mastery dict: {kc_id: row}
    - For each candidate topic and each of its prerequisites:
        - bucket = _mastery_bucket(prereq_mastery.get(prereq_id).p_mastery if row else None)
        - Skip if bucket is "in_progress" or "solid" (D-13)
        - Append topic.name to prereq_to_unlocks[prereq_id] where prereq_to_unlocks is dict[str, list[str]] keyed by prereq_id (NOT prereq_name)
    - Build output list: [{"prereq_name": _by_id[pid].name, "prereq_kc_id": pid, "unlocks": unlocks[:3]} for pid, unlocks in prereq_to_unlocks.items()] (D-12 cap at 3)
    - Return format: {"prereq_name": str, "prereq_kc_id": str, "unlocks": list[str]}
    - Return empty list if no candidates or no prereqs

    Function 6 — get_session_prereq_state(child_id: str) -> dict[str, int]:
    - Returns dict of {kc_id: turn_count} for all active escalation entries for this child
    - Implementation: {kc_id: count for (cid, kc_id), count in _prereq_turn_counter.items() if cid == child_id}

    Function 7 — extract_interests_from_turns(turns: list) -> list[str]:
    - Pure function (no DB); accepts list of objects with .answer attribute
    - Uses _tag_to_topic module-level index
    - For each turn, lowercases turn.answer (or "" if None)
    - For each tag in _tag_to_topic: if tag in answer_lower, tag_hits[tag] += 1
    - Collect topic names where any tag hit >= 2; return as list
    - D-09: case-insensitive substring, no NLP

    Function 8 — async extract_and_update_interests(session_id: str, child_id: str, db: AsyncSession) -> None:
    - Imports get_turns_by_session_id from db.crud
    - Calls `turns = await get_turns_by_session_id(session_id, db)`
    - If not turns: return
    - Calls `new_interests = extract_interests_from_turns(turns)`
    - If not new_interests: return
    - Calls `await update_interests(child_id, new_interests, db)` (D-10: set union, idempotent)
