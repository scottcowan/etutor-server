<objective>
Extend services/knowledge_tracing.py to implement the CURR-04 supersedes unlock check inside next_topics().

D-14 resolution (per planning context): `bloom_target` is a Bloom integer (1-6) while `p_mastery` is a BKT probability (0.0-1.0) — these scales are incommensurable. "Reaches bloom_target mastery" means the child has solidly mastered the topic per BKT. Use `p_mastery >= 0.95` (the existing `solid` bucket threshold) as the unlock trigger. Document this mapping in the code.

Purpose: When a child has solidly mastered a KC, any topic in the curriculum that supersedes that KC automatically becomes a candidate in next_topics(), ensuring model progression unlocks automatically.
Output: _superseded_by index + inject block + passing TDD tests.
</objective>

    ## GREEN Phase — Implement in services/knowledge_tracing.py

    Add module-level reverse index immediately after the CURRICULUM import (at module scope, before any function definition):

        # Reverse supersedes index: {superseded_topic_id: [superseding_topic_id, ...]}
        # Built once at import time — O(N) over CURRICULUM (870 topics, 14 have supersedes).
        # D-14: unlock trigger is p_mastery >= 0.95 (solid bucket threshold).
        # bloom_target is a Bloom integer (1-6); p_mastery is 0.0-1.0 — scales are incommensurable.
        # "Reaches bloom_target mastery" = solidly mastered per BKT = p_mastery >= 0.95.
        _superseded_by: dict[str, list[str]] = {}
        for _t in CURRICULUM:
            if _t.supersedes:
                _superseded_by.setdefault(_t.supersedes, []).append(_t.id)

    In next_topics(), immediately after the line that builds mastered_ids (the `p_mastery >= 0.95` list), and after the call to curriculum_next_topics() that produces `candidates`, add:

        # D-14: inject supersedes unlocks for fully-mastered (solid) KCs
        for kc_id in mastered_ids:
            for superseding_id in _superseded_by.get(kc_id, []):
                superseding_topic = _by_id.get(superseding_id)
                if superseding_topic and superseding_id not in mastered_ids:
                    if superseding_topic not in candidates:
                        candidates.append(superseding_topic)

    Placement is critical: this block runs AFTER curriculum_next_topics() returns `candidates` but BEFORE the final sort/limit applied to candidates. This ensures supersedes topics participate in the same ranking logic.
